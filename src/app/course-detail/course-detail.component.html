What is Angular?

Angular is a TypeScript-based frontend framework for building dynamic web applications. It is maintained by Google and a community of developers. You can use Angular to create single-page applications (SPAs) or complex web apps with rich features.

What Angular Does

It's a Framework: Angular is a full-fledged framework, unlike React, which is just a library. It provides everything you need to build a robust web application, including tools for routing, forms, HTTP requests, and more.

Developed by Google: Angular is developed and maintained by Google, ensuring it's backed by an active community and enterprise support.

Component-Based Development: Like React, Angular is also component-based. This means you build your app using reusable and standalone components that have their own logic and templates.

Modular: Angular is designed around modules, which help organize and structure an application. This helps in managing dependencies and keeping your codebase clean.

Awesome Features of Angular

  • Component-Based Architecture: Angular uses a component-based architecture, meaning you break down your application into components that represent different parts of the UI.
  • Two-Way Data Binding: Angular’s two-way data binding automatically synchronizes data between the model and the view. When the model changes, the view updates, and vice versa.
  • Fast: Angular provides high performance, especially in complex applications, by using efficient change detection and other optimizations.
  • Dependency Injection: Angular’s dependency injection system allows you to manage and inject services and components into each other, promoting modularity and testing.
  • TypeScript: Angular uses TypeScript, which adds static typing to JavaScript. TypeScript helps developers catch errors early and improves code readability.

Some Handy Terms

npm (Node Package Manager): npm is a package manager for JavaScript and TypeScript libraries, and Angular uses it to install and manage dependencies.

ng (Angular CLI): The Angular CLI is a command-line interface that simplifies development tasks like creating components, services, and running a local server for development.

🤔 What is JavaScript?

JavaScript is a dynamic programming language, a scripting language used to develop web applications, games, and more. It allows us to implement dynamic features on web pages that cannot be done with just HTML and CSS. JavaScript is supported by all browsers.

Brendan Eich invented JavaScript in 1995.

Where does JavaScript run?

JavaScript mainly runs in web browsers that have built-in engines to interpret and execute JavaScript code. JavaScript isn't just for web browsers; it also runs in Node.js, which provides a runtime environment that allows JavaScript to run.

JavaScript Engines with Different Browsers

  • Chrome uses V8 JavaScript engine and it is known for speed and efficiency.
  • Safari uses JavaScriptCore engine, developed by Apple.
  • Mozilla Firefox uses SpiderMonkey JavaScript engine, developed by the Mozilla Foundation.
  • Microsoft Edge uses the Chakra JavaScript engine, but with its transition to Chromium, it now also relies on the V8 engine.

Ternary Operator

The ternary operator, also known as the conditional operator, is an easy way to write simple conditional expressions. It provides a concise way to evaluate a condition and return one of two values based on whether the condition is true or false.

Syntax

Syntax:

(condition) ? expressionIfTrue : expressionIfFalse

How it works:

  1. The condition is evaluated.
  2. If the condition is true, the value of expressionIfTrue is returned.
  3. If the condition is false, the value of expressionIfFalse is returned.

Example 1

Check if a number is even or odd:


        let num = 10;
        num % 2 == 0 ? console.log('Even') : console.log('Odd');

Output: Even

Example 2

Check if a number is even or odd:


        let num = 7;
        num % 2 == 0 ? console.log('Even') : console.log('Odd');

Output: Odd

Introduction To React JS

React is a JavaScript frontend library for building user interfaces. It is maintained by Facebook and a community of individual developers and companies. You can use React to make single-page websites or mobile apps, which can be pretty cool!

What React Does

  • Developed by Facebook (Meta): Yep, smart folks at Facebook created React and continue to improve it.

How to Create a React App

There are primarily two best ways to create a new React app:

  1. Using CRA (create-react-app).
  2. Using Vite.

Both tools are popular and offer unique advantages for different project needs.

1. Using create-react-app

It was created and is maintained by Facebook's React team.

It uses Webpack internally.

Steps to Create a React App using CRA:

  1. Ensure that you have installed node.js on your machine.
  2. Open terminal or Git Bash, then type npx create-react-app (name) (e.g., npx create-react-app foody).
  3. It will take some time to create the React app.
  4. It will install some packages: react, react-dom, and react-scripts.
  5. Close the terminal and open that folder in VS Code.
  6. Open the terminal in VS Code.
  7. You will see all the necessary folders and files created.
  8. It will redirect to your default browser.
  9. Write this command: npm start. This command is used to start the development server.
  10. Open package.json file; you will see it installed lots of extra libraries like @testing-library/jest-dom, @testing-library/react, @testing-library/user-event, etc.
  11. Run npm run build to create a production-ready build of the application.

2. Using Vite

Vite is an open-source project created by Evan You, the creator of Vue.js.

It's a modern build tool that aims to provide a faster and more efficient development experience for frontend developers.

It has lightning-fast HMR (Hot Module Reloading) and comes with lots of features and plugins.

Steps to Create a React App using Vite:

  1. Open terminal and type npm create vite@latest.
  2. Next, it will ask for a name to keep.
  3. Next, it will ask to choose a framework; choose React (use arrow keys to navigate).
  4. It will ask to choose a variant; choose your favorite variant.
  5. Close the terminal and open that folder in VS Code.
  6. Open the terminal in VS Code and type npm i to install all the necessary libraries and dependencies.

Vite Folder Structure

Vite package.json File

Development Server

The default behavior of npm run dev is to start the development server using Vite. You can modify this behavior by editing the package.json file. In the scripts section, you'll find the first script labeled "dev": "vite". You can change it to "start" if you want to use a different command for development.

To start your development server, run npm run dev. The server will start on a port different from the CRA default (http://localhost:5173/). It won't automatically redirect you to the browser, so you'll need to click on the link displayed in the terminal after running the command to navigate there.

To create an optimized production build of your React application, run the build command: npm run build. This command will generate a dist folder containing your production-ready files.

After that, you are good to go for deployment.
